## [로그인] (https://staging.ailex.co.kr)

### 정상 상태 정의
- 이메일·비밀번호 입력 후 로그인 성공 시 홈(/) 로 이동
- 입력 필드 포커스 시 테두리 강조 표시
- 로그인 중 로딩 스피너 표시

### 의도된 제한사항
- 비로그인 상태에서 로그인 전용 경로(/case-list 등) 접근 시 로그인 화면으로 리다이렉트
- 잘못된 자격증명 5회 이상 실패 시 계정 잠금 또는 재시도 제한 가능 (정책 확인 필요)

### 비정상 케이스
- 잘못된 비밀번호 → 에러 메시지 표시 (화면 유지)
- 이메일 미입력 또는 형식 오류 → 유효성 에러 메시지
- 비밀번호 미입력 → 유효성 에러 메시지
- 네트워크 오류 → 서버 오류 안내

### 성공 기준
- 유효한 계정으로 로그인 시 2초 이내 홈 화면으로 이동
- 잘못된 자격증명 시 명확한 에러 메시지 표시

### 실패 기준
- 로그인 성공인데 홈 미이동
- 에러 메시지 없이 화면이 멈춤
- 비밀번호 평문 노출

## [홈 (사건목록)] (https://staging.ailex.co.kr/)

### 정상 상태 정의
- 로그인 후 자동 진입 또는 / 직접 접속 시 사건 목록 표시
- 사건 항목 클릭 시 /case-list?civil_case_id=...&project_id=... 로 이동
- 사건이 없는 경우 빈 상태 UI (empty state) 표시

### 의도된 제한사항
- 어드민 계정으로 로그인 시 어드민 홈(AdminHomePage)으로 분기
- 다른 사용자의 사건은 표시되지 않음

### 비정상 케이스
- API 오류로 사건 목록 미표시 → 에러 상태 UI 또는 재시도 안내
- 사건 목록 로드 지연 시 로딩 스켈레톤 표시

### 성공 기준
- 사건 목록이 3초 이내 표시됨
- 사건 클릭 시 /case-list로 정상 이동

### 실패 기준
- 로그인 후 홈 미표시 (빈 화면)
- 사건 클릭 시 무반응

## [민사 사건 메인 — 기록 목록 탭] (https://staging.ailex.co.kr/case-list?tab=list)

### 정상 상태 정의
- 사건에 속한 기록 문서가 테이블로 표시됨
- 문서명, 제출일, 제출자, 유형, 북마크/메모/클리핑 카운트 컬럼 표시
- 컬럼 헤더 클릭으로 정렬 전환 (asc/desc)
- 검색창 키워드 입력 시 실시간 필터링
- 문서 행 클릭 시 뷰어로 이동
- 페이지네이션 동작

### 의도된 제한사항
- 권한 없는 사건의 문서는 표시되지 않음
- 어드민 전용 필터/기능은 일반 사용자에게 미표시

### 비정상 케이스
- 문서 없음 → empty state UI 표시
- 검색 결과 없음 → '결과 없음' 안내
- 업로드 실패 → 오류 메시지 표시

### 성공 기준
- 문서 목록 5초 이내 로드
- 정렬·검색·페이지네이션 정상 동작
- 문서 클릭 시 뷰어로 정상 이동

### 실패 기준
- 목록 미표시 (스피너 무한)
- 검색 후 전체 목록이 사라짐
- 잘못된 문서로 이동

## [민사 사건 메인 — 문서 업로드] (기록 목록 탭 내 기능)

### 정상 상태 정의
- 문서 업로드 버튼 클릭 시 파일 선택 다이얼로그 오픈
- PDF 파일 선택 후 제목·문서유형·원고피고 여부 입력
- 업로드 진행 상태(progress) 표시
- 완료 시 목록 자동 갱신
- 파일명에 한글 포함 시 NFC 정규화 적용 후 업로드
- 다중 파일 업로드 지원 여부: 코드 단건 처리 기준

### 의도된 제한사항
- 지원 파일 형식: PDF (서버 정책에 따름)
- 파일 크기 제한: 서버 정책에 따름

### 비정상 케이스
- 제목 미입력 → 유효성 에러
- 지원 형식 외 파일 → 오류 안내
- 네트워크 단절 중 업로드 → 실패 메시지
- 서버 오류(5xx) → 재시도 안내
- 한글 파일명(NFC 미적용 환경) → 서버에서 인식 불가 예방

### 성공 기준
- 199KB PDF(sample_file.pdf) 업로드 30초 이내 완료
- 업로드 완료 후 기록 목록에 신규 문서 표시

### 실패 기준
- 업로드 완료 알림은 나오지만 목록에 미표시
- 업로드 중 화면 멈춤
- 한글 파일명 오작동

## [민사 사건 메인 — 자료 요청 탭] (https://staging.ailex.co.kr/case-list?tab=client_request)

### 정상 상태 정의
- 기존 요청 목록 표시
- 새 요청 생성 가능 (이메일, 자료명, 메시지 입력)
- 수신자 이메일 복수 추가 가능
- 요청 상태(제출 완료/미제출) 표시
- 첨부 파일 포함 요청 지원

### 의도된 제한사항
- 이미 제출 완료된 요청은 재발송 불가 (또는 별도 처리)
- 외부 이메일로 요청 발송 (내부 구성원 외)

### 비정상 케이스
- 이메일 형식 오류 → 유효성 에러
- 이메일 미입력 → 에러
- 자료명 미입력 → 에러
- 발송 실패 → 오류 메시지

### 성공 기준
- 요청 생성 후 목록에 표시
- 수신자 이메일로 발송 완료

### 실패 기준
- 요청 생성 후 목록 미표시
- 이메일 미발송

## [서면 작성 에디터] (https://staging.ailex.co.kr/case-list/editor)

### 정상 상태 정의
- TinyMCE 에디터 정상 로드 (바탕체 폰트 기본값)
- 텍스트 입력·편집 동작
- 자동 저장 동작
- 기록 목록 패널에서 문서/하이라이트 드래그앤드랍 → 증거 첨부
- 의뢰인 자료 패널에서 드래그앤드랍 지원
- 목차 열기/닫기 동작
- 증명 방법 명세 순서 조정 (가지번호 그룹화)
- 제출 정보 입력 (제출일, 담당 변호사)
- 문서명 변경 → 기록 목록·의뢰인 자료 연동
- 버전 기록 열람·복원
- 검토 요청 발송
- 다운로드 (DOCX 등)
- 메모 추가·댓글 (멘션 가능)

### 의도된 제한사항
- 현재 버전에서 복원 버튼 비활성화
- 검토 요청 후 메시지 수정 가능하나 삭제 불가
- 제출일 필드는 빈 값 또는 "2026. 3. " 같이 미완성 입력 허용
- 서면 탭 종료는 불가 (시간 기반 저장 설정)

### 비정상 케이스
- 에디터 로드 실패 → 오류 안내
- 자동 저장 실패 → 저장 실패 알림
- 드래그앤드랍 영역 외 드롭 → 무반응
- 다운로드 실패 → 오류 메시지
- 이름 변경 시 특수문자(/)·빈값 → 유효성 처리

### 성공 기준
- 에디터 5초 이내 로드
- 드래그앤드랍으로 증거 첨부 성공
- 버전 기록에 검토 요청 기록 저장
- 다운로드 파일 정상 생성

### 실패 기준
- 에디터 미로드 (빈 화면)
- 자동 저장 후 내용 유실
- 드래그앤드랍 동작 불가

## [의뢰인 증거 제출 요청 수신] (https://staging.ailex.co.kr/evidence-request)

### 정상 상태 정의
- 비로그인 상태에서 이메일 링크로 접근 가능
- 파일 선택 (PDF 등)
- 자료 이름 입력
- 제출 완료 시 완료 안내 화면

### 의도된 제한사항
- 만료된 요청 링크 → 유효하지 않음 안내
- 이미 제출된 요청 링크 → 중복 제출 방지 가능

### 비정상 케이스
- 파일 미선택 → 유효성 에러
- 자료명 미입력 → 유효성 에러
- 업로드 실패 → 오류 메시지

### 성공 기준
- 파일 업로드 및 제출 완료 안내 표시
- 담당 변호사 계정의 전체 자료 탭에 반영

### 실패 기준
- 제출 완료 후 목록 미반영
- 비로그인 접근 불가

### 실패 기준
