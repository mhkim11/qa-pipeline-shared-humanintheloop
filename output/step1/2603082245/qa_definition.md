## 로그인 (https://staging.ailex.co.kr)

### 정상 상태 정의
- 이메일·비밀번호 입력 폼이 표시된다
- 유효한 자격증명 입력 후 로그인 버튼 클릭 시 홈(/)으로 리다이렉트된다
- 로그인 후 세션이 유지되고 새로고침에도 로그인 상태가 유지된다
- 회원가입, 아이디 찾기, 비밀번호 재설정 링크가 동작한다

### 의도된 제한사항
- 비로그인 상태에서 로그인 전용 경로(/case-list, /evidence/* 등) 접근 시 로그인 화면으로 리다이렉트된다
- ADMIN 역할 사용자는 로그인 후 어드민 라우터로 분기된다

### 비정상 케이스
- 잘못된 이메일/비밀번호 입력 시 에러 메시지 표시
- 미가입 이메일로 로그인 시도 시 에러 메시지 표시
- 비밀번호 공란 제출 시 유효성 검증 오류 표시
- 네트워크 단절 시 오류 메시지 표시

### 성공 기준
- 유효한 계정으로 로그인 시 3초 이내 홈 화면으로 이동
- 로그인 후 URL이 /로 변경되고 홈 콘텐츠가 표시됨

### 실패 기준
- 유효 계정 로그인 후 홈으로 이동하지 않음
- 잘못된 자격증명에도 로그인 성공
- 로그인 화면이 렌더링되지 않음

## 홈 (사건목록) (https://staging.ailex.co.kr/)

### 정상 상태 정의
- 로그인 후 사건 목록(EvidenceListTable)이 표시된다
- 등록된 사건이 카드/리스트 형태로 나열된다
- 새 사건 등록 버튼이 표시되고 클릭 시 민사/형사 선택 모달이 열린다
- 좌측 네비게이션(사이드바)이 표시되고 각 메뉴 클릭 시 해당 화면으로 이동한다

### 의도된 제한사항
- 등록된 사건이 없을 경우 빈 상태 UI(Empty State)가 표시된다
- 권한이 없는 사건은 목록에 표시되지 않는다

### 비정상 케이스
- 사건 목록 API 실패 시 오류 메시지 표시
- 대용량 사건 목록(100건 이상) 로딩 시 스피너 표시
- 사건 데이터 없음(빈 필드) 시 레이아웃 깨짐 없이 표시

### 성공 기준
- 홈 진입 후 3초 이내 사건 목록이 표시됨
- 새 사건 등록 모달이 정상 열리고 닫힘
- 좌측 네비게이션 모든 메뉴가 동작함

### 실패 기준
- 홈 화면이 빈 화면으로 표시됨
- 사건 목록이 로딩되지 않음
- 새 사건 등록 버튼이 동작하지 않음

## 사건 목록 — 기록 목록 (https://staging.ailex.co.kr/case-list?...&tab=list)

### 정상 상태 정의
- 선택된 사건의 기록 문서 목록(CaseMainListTable)이 표시된다
- 문서 클릭 시 상세 뷰(CaseDetailListTable)로 전환된다
- 좌측 사이드바에서 기록 목록/의뢰인 자료/서면 작성 탭 전환이 동작한다
- 사이드바 접기/펼치기 버튼이 동작한다
- URL에 civil_case_id 또는 project_id 파라미터가 올바르게 포함된다

### 의도된 제한사항
- 사건 ID 없이 /case-list 접속 시 선택된 문서 없이 목록만 표시된다
- 레거시 파라미터(caseId, client_hidden)도 정상 처리된다

### 비정상 케이스
- 존재하지 않는 civil_case_id로 접속 시 빈 목록 또는 오류 메시지 표시
- 기록 목록 API 실패 시 오류 상태 표시
- 특수문자 포함 문서명 표시 시 레이아웃 깨짐 없음

### 성공 기준
- 기록 목록이 3초 이내 표시됨
- 문서 클릭 시 상세 뷰로 전환되고 URL에 case_document_id 파라미터 추가됨
- 탭 전환 시 URL 파라미터가 업데이트됨

### 실패 기준
- 기록 목록이 표시되지 않음
- 문서 클릭 후 화면 전환이 발생하지 않음
- 탭 전환 시 화면이 변경되지 않음

## 사건 목록 — 자료 요청 (https://staging.ailex.co.kr/case-list?...&tab=client_request)

### 정상 상태 정의
- 의뢰인에게 자료 요청을 생성하고 목록을 조회할 수 있다
- 요청 링크 복사 기능이 동작한다
- 제출 내역 확인이 가능하다

### 의도된 제한사항
- 사건 ID가 없는 상태에서 의뢰인 자료 탭 클릭 시 이동하지 않는다 (코드: `if (!selectedCaseId) return;`)

### 비정상 케이스
- 요청 생성 API 실패 시 오류 메시지 표시
- 빈 요청 목록 시 Empty State 표시

### 성공 기준
- 자료 요청 목록이 표시됨
- 링크 복사 기능이 클립보드에 정상 복사됨

### 실패 기준
- 자료 요청 탭 클릭 후 화면이 변경되지 않음
- 링크 복사 실패

## 사건 목록 — 서면 작성 탭 (https://staging.ailex.co.kr/case-list?...&tab=editor)

### 정상 상태 정의
- 서면 목록(CaseDocumentEditorWrapper)이 표시된다
- 서면 항목 클릭 또는 새 서면 작성 버튼으로 에디터 화면(/case-list/editor)으로 이동한다
- 탭 전환 시 URL이 tab=editor로 업데이트된다

### 의도된 제한사항
- 사건 ID 없이 서면 작성 탭 접근 가능하지만 서면 목록이 비어있을 수 있다

### 비정상 케이스
- 서면 목록 API 실패 시 오류 메시지 표시
- 서면 제목에 특수문자(예: < > & ") 포함 시 XSS 없이 정상 표시

### 성공 기준
- 서면 목록이 표시됨
- 에디터로의 이동이 정상 동작함

### 실패 기준
- 서면 목록이 표시되지 않음
- 에디터로 이동하지 않음

## 서면 작성 에디터 (https://staging.ailex.co.kr/case-list/editor?...)

### 정상 상태 정의
- 전체화면 에디터(DocumentEditorView)가 표시된다
- AI 서면 생성 기능이 동작한다 (AI 분석 결과 기반 초안 생성)
- 텍스트 입력, 서식 적용(볼드/이탤릭 등)이 동작한다
- 저장 버튼 클릭 시 서면이 저장된다
- 뒤로가기 버튼 클릭 시 /case-list?tab=editor로 복귀한다

### 의도된 제한사항
- 에디터 이탈 시 미저장 경고 없이 이탈 가능할 수 있음 (UX 확인 필요)
- civil_case_id/project_id 파라미터 없이 접근 시 에디터가 제한될 수 있음

### 비정상 케이스
- AI 서면 생성 API 실패 시 오류 메시지 표시
- 대용량 텍스트(5000자 이상) 입력 시 성능 저하 없음 확인
- 빈 서면 저장 시도 시 유효성 검증 오류 표시
- 특수문자 및 이모지 포함 텍스트 저장/불러오기 정상 처리

### 성공 기준
- 에디터가 3초 이내 로딩됨
- AI 서면 생성 후 에디터에 텍스트가 삽입됨
- 저장 후 성공 메시지 또는 상태 변경이 표시됨
- 뒤로가기 후 URL이 /case-list?tab=editor로 복귀됨

### 실패 기준
- 에디터가 로딩되지 않음
- AI 생성 기능이 동작하지 않음
- 저장이 되지 않음

## 증거 목록 (https://staging.ailex.co.kr/evidence/list)

### 정상 상태 정의
- 증거 문서 목록(EvidenceTable)이 표시된다
- 검색 입력 시 필터링이 동작한다
- 문서 클릭 시 PDF 뷰어(/evidence/pdf/:id)로 이동한다
- 업로드 버튼으로 새 증거 문서를 추가할 수 있다
- 히스토리, 권한 탭으로 전환이 가능하다

### 의도된 제한사항
- 권한이 없는 문서는 조회할 수 없다
- 지원하지 않는 파일 형식 업로드 시 오류 메시지 표시

### 비정상 케이스
- 증거 목록 API 실패 시 오류 상태 표시
- 빈 목록 시 Empty State 표시
- 파일명에 특수문자/한글 포함 시 정상 표시
- 대용량 파일(100MB 이상) 업로드 시 처리 방식 확인

### 성공 기준
- 증거 목록이 3초 이내 표시됨
- 검색 시 즉각적인 필터링 또는 API 호출이 이루어짐
- 파일 업로드 후 목록에 신규 항목이 추가됨

### 실패 기준
- 증거 목록이 표시되지 않음
- 검색 기능이 동작하지 않음
- 파일 업로드가 실패함

## 증거 PDF 뷰어 (https://staging.ailex.co.kr/evidence/pdf/:evidenceId)

### 정상 상태 정의
- PDF 파일이 iframe으로 정상 렌더링된다
- 간편검색 버튼 클릭 시 드래그 가능한 텍스트 검색 모달이 열린다
- 모달에서 Ctrl+F/Cmd+F로 텍스트 검색이 가능하다
- 모달 닫기(X) 버튼이 동작한다
- 모바일에서는 서버 URL 직접 사용 또는 blob URL 폴백이 동작한다

### 의도된 제한사항
- evidenceId 또는 projectId 파라미터 없이 접근 시 PDF가 로딩되지 않는다 (loading=false, PDF 미표시)
- 페이지 이탈 시 "페이지를 떠나시겠습니까?" 확인 다이얼로그가 표시된다
- 확인 선택 시 탭/창이 닫힌다

### 비정상 케이스
- 존재하지 않는 evidenceId로 접속 시 로딩 실패 메시지 표시
- PDF API 응답 지연(3초 이상) 시 스피너 표시
- 빈 PDF(0바이트) 처리 시 오류 없이 빈 화면 표시 또는 안내 메시지
- 모바일에서 서버 URL 실패 시 blob URL로 폴백 동작 확인

### 성공 기준
- PDF가 5초 이내 렌더링됨
- 간편검색 모달이 정상 열리고 텍스트가 표시됨
- 모달이 화면 경계 내에서 드래그 가능함

### 실패 기준
- PDF가 렌더링되지 않음
- 간편검색 버튼이 동작하지 않음
- 모달이 표시되지 않거나 텍스트가 비어있음

## AI 분석 (https://staging.ailex.co.kr/ai)

### 정상 상태 정의
- 좌측 패널에 project_id 기반 AI 분석 카테고리·메뉴가 로딩된다
- 메뉴 항목 클릭 시 우측 콘텐츠 영역에 HTML 또는 Markdown 리포트가 렌더링된다
- 첫 번째 메뉴가 자동 선택되어 콘텐츠가 표시된다
- Markdown 파일은 marked + DOMPurify로 안전하게 렌더링된다

### 의도된 제한사항
- project_id 파라미터가 없으면 분석 메뉴가 표시되지 않거나 빈 상태로 표시된다
- 카테고리별 활성화(isEnabled) 여부에 따라 표시 여부가 달라진다

### 비정상 케이스
- 분석 메뉴 API 실패 시 "메뉴를 불러오는 중 오류가 발생했습니다" 표시
- 메뉴 로딩 중에는 "메뉴 로딩중..." 스피너 표시
- 파일 URL이 잘못된 경우 "파일을 불러오는 중 오류가 발생했습니다" 표시
- XSS 공격 패턴 포함 MD 파일 → DOMPurify로 sanitize 후 렌더링 (script 태그 제거 확인)

### 성공 기준
- AI 메뉴가 3초 이내 표시됨
- 메뉴 클릭 시 리포트 콘텐츠가 5초 이내 표시됨
- Markdown이 HTML로 정상 변환되어 표시됨

### 실패 기준
- 메뉴가 표시되지 않음
- 메뉴 클릭 후 콘텐츠가 표시되지 않음
- 에러 메시지 없이 빈 화면 표시

## 설정 (https://staging.ailex.co.kr/settings)

### 정상 상태 정의
- MainHeader와 SettingTable이 표시된다
- 계정 정보(이메일, 닉네임, 사무소명 등)를 조회할 수 있다
- 설정값 변경 후 저장이 가능하다

### 의도된 제한사항
- 이메일 변경은 인증 절차가 필요할 수 있다
- 일부 설정은 ADMIN 역할에서만 변경 가능할 수 있다

### 비정상 케이스
- 설정 저장 API 실패 시 오류 메시지 표시
- 유효하지 않은 값 입력(예: 이메일 형식 오류) 시 유효성 오류 표시

### 성공 기준
- 설정 화면이 3초 이내 로딩됨
- 변경 사항 저장 후 성공 메시지 또는 반영 확인

### 실패 기준
- 설정 화면이 표시되지 않음
- 저장이 동작하지 않음

### 성공 기준
- 결제 화면이 3초 이내 표시됨
- 결제 수단 추가 후 목록에 반영됨

### 실패 기준
- 결제 화면이 표시되지 않음
- 결제 정보가 마스킹 없이 노출됨

## 증거 제출 요청 — 외부 의뢰인 (https://staging.ailex.co.kr/evidence-request)

### 정상 상태 정의
- 비로그인 상태에서 접근 가능하다
- 파일 업로드 UI가 표시된다
- 파일 선택 후 제출 버튼 클릭 시 업로드가 완료된다
- 제출 완료 후 확인 메시지가 표시된다

### 의도된 제한사항
- 유효한 요청 링크(토큰 또는 파라미터)가 없으면 접근이 제한될 수 있다
- 지원하지 않는 파일 형식 업로드 시 오류 메시지 표시

### 비정상 케이스
- 빈 파일 제출 시 유효성 검증 오류 표시
- 대용량 파일(100MB 이상) 업로드 시 처리 방식 확인
- 특수문자 포함 파일명 업로드 시 정상 처리
- 네트워크 단절 중 업로드 시 오류 처리

### 성공 기준
- 파일 업로드 후 서버에 정상 저장됨
- 제출 완료 메시지 표시됨
- 변호사 측 자료 요청 탭에서 제출 내역 확인 가능

### 실패 기준
- 파일 업로드 실패
- 제출 완료 후 확인 메시지 없음
- 제출된 파일이 변호사 측에서 조회되지 않음
