## 로그인 (/)

### 정상 상태 정의
- 이메일/비밀번호 입력 필드가 표시된다
- 유효한 자격증명 입력 시 홈 화면으로 이동한다
- 로그인 상태 유지 (세션/토큰 저장)

### 의도된 제한사항
- 비로그인 상태에서 다른 경로 직접 접근 시 로그인 화면으로 리다이렉트
- 5회 이상 로그인 실패 시 일시 잠금

### 비정상 케이스
- 잘못된 이메일 형식 입력 시 유효성 오류 메시지 표시
- 틀린 비밀번호 입력 시 "이메일 또는 비밀번호가 올바르지 않습니다" 메시지 표시
- 네트워크 오류 시 재시도 안내 표시

### 성공 기준
- 유효한 계정으로 로그인 후 홈 화면 진입 성공

### 실패 기준
- 유효한 계정으로 로그인 실패
- 로그인 후 홈이 아닌 다른 화면으로 이동
- 오류 메시지 미표시

## 사건목록 (/case-list)

### 정상 상태 정의
- 사건 목록이 카드 또는 테이블 형태로 표시된다
- 탭(목록 / 의뢰인 요청 / 의뢰인 목록 / 편집자) 전환이 동작한다
- 각 사건 항목 클릭 시 해당 상세로 이동한다

### 의도된 제한사항
- 편집자 권한 없는 사용자는 서면 작성 에디터 버튼 비활성화

### 비정상 케이스
- 사건 데이터 없을 시 빈 상태 안내 메시지 표시
- API 오류 시 에러 메시지 표시

### 성공 기준
- 4개 탭 전환 정상 동작
- 사건 항목 클릭 시 정상 이동

### 실패 기준
- 탭 클릭 시 내용 변경 없음
- 사건 항목 클릭 시 오류 발생

## 서면 작성 에디터 (/case-list/editor)

### 정상 상태 정의
- 리치 텍스트 에디터가 로드된다
- 텍스트 입력, 서식 적용(굵게/기울임/밑줄)이 동작한다
- 저장 버튼 클릭 시 POST /api/documents 호출 후 성공 메시지 표시

### 의도된 제한사항
- 편집자 권한 없는 사용자 접근 차단 → 403 또는 리다이렉트

### 비정상 케이스
- 빈 문서 저장 시도 시 경고 메시지 표시
- 저장 중 네트워크 오류 시 임시저장 안내

### 성공 기준
- 텍스트 입력 후 저장 성공 토스트 표시
- 페이지 재진입 시 저장된 내용 유지

### 실패 기준
- 저장 버튼 클릭 후 오류 발생
- 저장 후 내용이 초기화됨
