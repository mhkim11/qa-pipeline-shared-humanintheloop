## [로그인] (/)

### 정상 상태 정의
- 이메일 + 비밀번호 입력 폼이 표시된다
- 이메일 형식 실시간 유효성 검사가 동작한다 (zod)
- 비밀번호 필드 포커스 시 이메일 존재 여부 API 호출이 발생한다
- '아이디 저장' 체크박스 선택 시 localStorage에 이메일이 저장된다
- 페이지 재방문 시 저장된 이메일이 자동 입력된다
- 로그인 성공 시 홈 화면(/)으로 이동한다
- 비밀번호 표시/숨김 토글이 정상 동작한다

### 의도된 제한사항
- 이메일 형식 불일치 시 로그인 버튼 비활성화
- 비밀번호 미입력 시 로그인 버튼 비활성화
- 로그인 실패(잘못된 비밀번호) 시 오류 메시지 표시, 페이지 유지
- 로그인 성공 후 로그아웃 경로(/logout)는 홈으로 리다이렉트

### 비정상 케이스
- 서버 오류(5xx) 시 오류 메시지 표시
- 이메일 존재 여부 체크 API 실패 시 폼 정상 동작 유지 (graceful degradation)

### 성공 기준
- 유효한 계정으로 로그인 시 홈(/)으로 이동
- 아이디 저장 기능 정상 동작
- 유효성 검사 오류 시 버튼 비활성화 확인

### 실패 기준
- 유효한 계정인데 로그인 실패
- 로그인 후 홈이 아닌 다른 페이지로 이동
- 비밀번호 노출 상태로 화면 표시

## [회원가입 — 약관동의] (/register)

### 정상 상태 정의
- 서비스 이용약관, 개인정보처리방침, 마케팅 동의 3가지 체크박스 표시
- '전체 동의' 체크박스 선택 시 3가지 모두 자동 선택
- 필수 약관(서비스, 개인정보) 동의 후 본인인증 버튼 활성화
- NICE 본인인증 팝업(데스크톱) 또는 리다이렉트(모바일) 정상 실행

### 의도된 제한사항
- 필수 약관(서비스 이용약관, 개인정보처리방침) 미동의 시 다음 단계 진행 불가
- 24시간 내 5회 초과 인증 시도 시 차단 모달 표시
- 마케팅 동의는 선택 사항

### 비정상 케이스
- NICE API 호출 실패 시 오류 메시지 표시
- 팝업 차단 환경에서 리다이렉트로 폴백

### 성공 기준
- 필수 약관 동의 후 본인인증 완료 시 이메일 등록 단계로 이동
- 마케팅 동의 여부가 localStorage에 정상 저장

### 실패 기준
- 필수 약관 미동의 상태로 다음 단계 진행 가능한 경우
- 인증 횟수 초과 시 차단되지 않는 경우

## [홈 — 증거 목록 메인] (/)

### 정상 상태 정의
- 로그인 후 기본 탭으로 EvidenceTable 표시
- 헤더에서 히스토리, 권한관리, AI, 결제, 구독 탭 전환 가능
- URL `?active=` 파라미터로 탭 직접 접근 가능
- 각 탭 전환 시 해당 컴포넌트 즉시 렌더링

### 의도된 제한사항
- 비로그인 상태에서 접근 시 로그인 페이지로 이동
- 일반 사용자는 어드민 전용 기능 접근 불가

### 비정상 케이스
- 탭 전환 시 데이터 로딩 실패 → 오류 메시지 표시
- EvidenceTable 데이터 조회 실패 → 빈 목록 또는 오류 표시

### 성공 기준
- 모든 탭 전환이 정상 동작
- URL 파라미터로 탭 직접 접근 가능

### 실패 기준
- 탭 전환 후 이전 탭 컴포넌트가 남아 있는 경우
- `?active=` 파라미터 무시되는 경우

## [민사 사건 — 기록 목록 탭] (/case-list?tab=list)

### 정상 상태 정의
- civil_case_id 또는 project_id 파라미터 기반 문서 목록 조회
- CaseMainListTable 렌더링, 문서 클릭 시 CaseDetailListTable로 전환
- 검색 기능: 검색어 입력 시 실시간 필터링 또는 API 호출
- 사이드바 '기록 목록' 클릭 시 목록으로 복귀 (replace 히스토리)
- 브라우저 뒤로가기로 문서 목록 복귀 가능

### 의도된 제한사항
- civil_case_id/project_id 없으면 기본 목록 표시(전체 또는 안내 메시지)
- 문서 선택 상태에서 URL에 case_document_id 파라미터 추가

### 비정상 케이스
- API 실패 시 빈 목록 또는 오류 메시지
- 잘못된 civil_case_id 입력 시 빈 목록

### 성공 기준
- 문서 목록 정상 표시
- 문서 클릭 시 상세 보기 전환
- 검색 기능 정상 동작

### 실패 기준
- 문서 목록 미표시
- 문서 클릭 후 화면 전환 미발생

## [민사 사건 — 서면 작성 탭] (/case-list?tab=editor)

### 정상 상태 정의
- CaseDocumentEditorWrapper 렌더링
- 서면 목록 표시 (상태: 생성 준비/생성중/작성중/제출됨)
- '서면 생성하기' 버튼 클릭 시 생성 폼 표시
- 서면 더보기 메뉴: 상태에 따라 이름 바꾸기/다른이름으로저장/다운로드/삭제 표시

### 의도된 제한사항
- 생성 준비 상태: 이름 바꾸기, 다른이름으로저장, 삭제만 가능
- 제출됨 상태: 다른이름으로저장 버튼 disabled
- 생성중 상태: 모든 메뉴 disabled

### 비정상 케이스
- 서면 생성 API 실패 시 오류 메시지
- 서면 목록 조회 실패 시 빈 목록

### 성공 기준
- 서면 목록 정상 표시
- 상태별 더보기 메뉴 정책 준수
- 서면 클릭 시 에디터 진입

### 실패 기준
- 제출됨 상태인데 다른이름으로저장 활성화
- 생성중 상태인데 메뉴 활성화

## [서면 작성 에디터] (/case-list/editor)

### 정상 상태 정의
- TinyMCE 에디터 로딩 및 서면 내용 표시
- 자동 저장: 검토요청 시, 서면 최초 생성 시, 나가기 시, 버튼 클릭 시 버전 저장
- 증거 첨부: 기록 목록/의뢰인 자료 드래그앤드롭 또는 클릭 첨부
- 검토 요청 모달: 메시지 입력 → 전송 → 버전 기록 생성
- 버전 기록: 이전 버전과 Diff 표시, 복원 버튼
- 디테일 수정: 제출일, 담당 변호사 등 직접 입력
- 메모: 텍스트 드래그 후 메모 추가, 맨션 가능, 댓글 가능
- 목차: 목차 열기/닫기 토글

### 의도된 제한사항
- 현재 버전에서 버전 복원 버튼 비활성화
- 메시지 삭제 불가 (수정만 가능)
- 서면 에디터 폰트: 바탕체 고정
- `/case-list/editor` 탭 종료 시 버전 저장 불가 (시간 기반만 가능)

### 비정상 케이스
- 에디터 로딩 실패 시 오류 메시지
- 검토 요청 API 실패 시 오류 토스트
- 자동 저장 실패 시 경고 표시

### 성공 기준
- 에디터 정상 로딩 및 내용 편집 가능
- 검토 요청 전송 시 버전 기록 생성
- 증거 첨부 시 증명 방법 명세에 반영
- 목차·메모·버전기록·디테일수정 패널 열기/닫기 정상

### 실패 기준
- 에디터 미로딩
- 검토 요청 후 버전 기록 미생성
- 자동 저장 미동작

## [증거 목록] (/evidence/list)

### 정상 상태 정의
- 증거 문서 목록 표시 (파일명, 상태, 날짜, 페이지수 등)
- 파일 업로드: PDF 전용, 500MB 이하, 다중 선택 가능
- 검색·필터·정렬 기능 동작
- 문서 행 클릭 시 뷰어 이동
- 다운로드 버튼 클릭 시 파일 다운로드
- XLSX 내보내기 기능 (증거목록.xlsx)

### 의도된 제한사항
- PDF 외 파일(HWP, DOCX 등) 업로드 차단
- 500MB 초과 파일 업로드 차단
- HWP/HWPX 파일 명시적 차단 메시지 제공

### 비정상 케이스
- 업로드 실패 시 파일 항목에 오류 상태 표시
- 500MB 초과 시 파일 항목에 `sizeExceeded` 상태 표시

### 성공 기준
- 정상 PDF 업로드 완료 및 목록에 즉시 반영
- 형식 오류·용량 초과 시 명확한 메시지 표시
- 다운로드 정상 동작

### 실패 기준
- HWP 파일이 업로드되는 경우
- 500MB 초과 파일이 업로드되는 경우
- 업로드 오류 메시지 미표시

## [증거 PDF 뷰어] (/evidence/pdf/:evidenceId)

### 정상 상태 정의
- PDF 파일 iframe 또는 직접 렌더링

- PDF 로딩 실패 시 오류 메시지
- 존재하지 않는 evidenceId 시 404 또는 오류

### 성공 기준
- PDF 정상 렌더링
- 인쇄 기능 동작

### 실패 기준
- PDF 로딩 실패 후 오류 미표시
- 잘못된 evidenceId에서 빈 화면만 표시

## [AI 분석] (/ai)

### 정상 상태 정의
- project_id 파라미터 기반 AI 분석 메뉴 API 조회
- 카테고리별 메뉴 목록 왼쪽 패널에 표시
- 메뉴 클릭 시 HTML(iframe) 또는 MD(marked 렌더링) 파일 오른쪽 표시
- 첫 번째 메뉴 자동 선택
- 선택 메뉴 강조 표시 (파란색 텍스트 + bold)

### 의도된 제한사항
- project_id 없으면 빈 메뉴 또는 오류
- 로그인 필수

### 비정상 케이스
- 메뉴 API 실패 시 '메뉴를 불러오는 중 오류가 발생했습니다.' 표시
- 파일 로딩 실패 시 빨간 오류 텍스트
- 파일 디코딩 실패 시 '[디코딩 실패]' 표시

### 성공 기준
- 메뉴 로딩 완료 및 카테고리·메뉴 표시
- 메뉴 클릭 시 해당 파일 렌더링
- HTML/MD 두 형식 모두 정상 표시

### 실패 기준
- 메뉴 로딩 실패 후 오류 미표시
- 파일 렌더링 실패 후 화면 전환 없음

## [어드민 메인] (/) — adminRouter

### 정상 상태 정의
- 어드민 계정(role: ADMIN) 로그인 시 adminRouter 적용
- AdminHomePage 렌더링
- 어드민 대시보드 표시

### 의도된 제한사항
- 일반 사용자(role != ADMIN)는 userRouter 적용, 어드민 라우터 접근 불가

### 비정상 케이스
- 일반 사용자가 /admin/* 경로 직접 접근 시 접근 거부 또는 리다이렉트

### 성공 기준
- 어드민 계정으로 로그인 시 AdminHomePage 표시
- 일반 사용자로 로그인 시 AdminHomePage 표시 안됨

### 실패 기준
- 일반 사용자가 어드민 기능에 접근 가능한 경우

## [의뢰인 자료 제출] (/evidence-request)

### 정상 상태 정의
- 비로그인 상태에서도 접근 가능 (의뢰인 전용)
- 파일 첨부: PDF, DOC, DOCX, 이미지 파일 허용
- 500MB 이하 파일만 허용
- 다중 파일 업로드 가능
- 기타자료 제출 모달 열기/닫기/확인 정상 동작

### 의도된 제한사항
- HWP/HWPX 파일 업로드 차단
- 500MB 초과 파일 차단
- 로그인 없이 접근 가능 (의뢰인 특성)

### 비정상 케이스
- HWP 파일 선택 시 '한글(HWP/HWPX) 파일 업로드를 지원하지 않습니다.' 토스트
- 500MB 초과 시 크기 제한 초과 메시지 표시
- 지원하지 않는 형식: '지원하지 않는 파일 형식입니다.' 토스트

### 성공 기준
- 허용 형식 파일 정상 업로드
- 차단 형식·용량 초과 시 명확한 오류 메시지
- 기타자료 제출 모달 정상 동작

### 실패 기준
- HWP 파일이 업로드 처리되는 경우
- 오류 메시지 미표시
- 모달 열기/닫기 오작동
